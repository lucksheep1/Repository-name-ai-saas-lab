# No Signal Decision Rules - 无信号分叉决策规则

## 核心原则

**"无信号"不是继续小修小补的理由，而是必须触发一次明确、单选、带动作的路径分叉。**

---

## 无信号处理流程

```
读取信号 -> 判定级别 -> 单选分叉 -> 执行动作 -> 记录
```

---

## 判定规则表

| 信号级别 | 判定条件 | 单选分叉选项 | 本轮动作 |
|---------|---------|-------------|---------|
| 强信号 | comments >= 3 或 reactions >= 5 | 深入追打 | 围绕该主题连续发 Issue |
| 弱信号 | comments >= 1 或 reactions >= 1 | 加压同路径 | 优化 CTA/问题设计 |
| **无信号** | comments = 0 且 reactions = 0 | **四选一** | **必须选一个执行** |

---

## 无信号四选一分叉

当读到"无信号"时，必须从以下四个选项中选择**一个**：

### 1. 加压当前入口
- **条件**: 问题本身有价值，只是曝光不够
- **动作**: 重新发布、更强 CTA、更多平台同步
- **不做原因**: Issue #3 已经带投票+CTA，再加边际效益为0

### 2. 切换更强入口
- **条件**: 当前入口本身无法捕获信号
- **动作**: 切换到 Discussion / Twitter / Discord 等
- **不做原因**: Discussion 未启用

### 3. 改变问题结构
- **条件**: 问题设计不够吸引人
- **动作**: 重新设计问题、用更硬的验证（如付费意愿）
- **原因**: 做了！Issue #4 直接问付费意愿

### 4. 改变目标对象
- **条件**: 当前对象不匹配
- **动作**: 重新定位目标用户
- **不做原因**: 没有数据支撑新目标
