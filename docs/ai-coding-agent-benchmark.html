Why agent-memory Is the Missing Evaluation Metric

Most benchmarks measure coding capability — but forget that memory affects long-horizon task performance. A coding agent with poor memory fails at tasks requiring context from 100+ messages ago. agent-memory enables:

  • Persistent context — agent-memory survives session restarts, unlike in-context memory
  • TTL-based forgetting — benchmark-clean memory between tests
  • Cross-session continuity — measure agent performance over days/weeks, not just hours
  • MIT license — use in any benchmark harness without licensing constraints
