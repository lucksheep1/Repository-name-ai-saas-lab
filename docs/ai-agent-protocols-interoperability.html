The AI Agent Interoperability Problem

Every AI agent framework — LangChain, CrewAI, AutoGen, Mastra — is a silo. Agents can't talk to each other. A2A and ACP solve this: open protocols that let any agent communicate with any other agent, regardless of framework. agent-memory plugs into this ecosystem: memory is the shared state layer that A2A/ACP agents use to coordinate.

a2aproject/A2A: Agent2Agent Open Protocol

"Agent2Agent (A2A) is an open protocol enabling communication and interoperability between opaque agentic applications. Built for agent-to-agent communication regardless of framework." — a2aproject/A2A on GitHub

"The first protocol for multi-agent communication and coordination. Combine any set of state-of-the-art agents with ACP and access any set of contexts with MCP (Model Context Protocols) to create state-of-the-art multi-agent systems." — agent-context-protocol/agent-context-protocol on GitHub

agntcy/acp-sdk: Cross-Framework Agent SDK

"Agent Connect Protocol SDK — facilitating cross-framework agent interoperability. Built with support from the IoA community to enable agents from different frameworks to work together." — agntcy/acp-sdk on GitHub

Why agent-memory Is the Memory Layer for A2A/ACP

  • Shared state for multi-agent protocols — A2A/ACP agents need shared memory to coordinate
  • Framework-agnostic — works with any agent regardless of underlying framework
  • TTL + encryption — coordination state with automatic expiry
  • MCP-compatible — built on Model Context Protocol for maximum interoperability
  • MIT licensed — open protocol needs open memory infrastructure
