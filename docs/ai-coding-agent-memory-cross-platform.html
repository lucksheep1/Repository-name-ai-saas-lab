The Cross-Platform Memory Problem

Developers use multiple AI coding tools: Claude Code for terminal tasks, Cursor for IDE work, Windsurf for quick edits, Copilot for autocomplete. Each tool has its own memory — none of them share. memoir, memorix, memex, and hmem solve this.

"Persistent memory for AI coding tools. Sync Claude, Cursor, Gemini, Copilot, Windsurf + 6 more across machines via MCP. E2E encrypted. Open source." — camgitt/memoir on GitHub, 1 week ago

Why agent-memory is the Cross-Platform Memory Solution

  • Works with any MCP agent — Claude Code, Cursor, Cline, Windsurf, OpenClaw, OpenHands, Gemini CLI
  • AES-256 encryption — mandatory encryption, not optional
  • TTL auto-expiration — stale memories auto-delete, only memoir offers E2E encryption
  • Redis backend — shared memory across multiple agents and machines simultaneously
  • Three backends — JSON (simple), SQLite (persistent), Redis (shared)
  • MIT license — audit, modify, deploy commercially without restrictions
