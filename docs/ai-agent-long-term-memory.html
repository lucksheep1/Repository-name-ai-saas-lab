Long-Term Episodic+Semantic Cross-Session

The Long-Term Memory Problem

Short-term context windows forget everything between sessions. Today's AI agent conversation — the decisions made, the approaches that failed, the user preferences learned — vanishes when the session ends. Without long-term memory, every session starts from scratch. This is the core limitation holding back AI agents from genuine continuity and learning.

Why agent-memory Is Long-Term Memory Infrastructure

  • Cross-session persistence — Redis survives session restarts
  • TTL-based episodic memory — time-bounded experience storage
  • Semantic structured storage — key-value with metadata for facts
  • Hierarchical TTL — short TTL for working memory, long TTL for knowledge
  • AES-256 encryption — long-term memories are sensitive
  • MIT license — deploy long-term memory infrastructure without vendor lock-in
# agent-memory: long-term memory pip install agent-memory # Cross-session memory with TTL hierarchy python -m agent_memory.mcp_server \ --storage redis \ --encryption aes-256 \ --path ./long-term-memory \ --ttl 2592000 # Episodic: short TTL (24h) for experiences # Semantic: long TTL (30d) for facts # Working: medium TTL (1h) for active context # MIT licensed
