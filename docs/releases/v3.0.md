# v3.0 - SQLite + TTL Release

**Release Date:** 2026-03-17
**Status:** Latest Stable

---

## What's New

### SQLite Backend (Production Ready)
- Persistent storage with SQLite
- No file corruption risks
- Better performance for large memory datasets

### TTL Support (Automatic Expiration)
- Set default TTL for all memories
- Per-memory TTL override
- Automatic memory decay

### Tags & Priority
- Tag-based organization
- Priority levels (1-5)
- Filter by tag or priority

---

## Quick Start

```python
from agent_memory import Memory

# SQLite with 30-day TTL (recommended)
memory = Memory(storage="sqlite", path="./memory.db", ttl_days=30)

# Add memories
memory.add("User prefers dark mode", ttl_days=7)
memory.add_with_tags("Bug in login", tags=["bug", "urgent"])

# Search and retrieve
results = memory.search("theme preferences")
context = memory.get_context(max_tokens=2000)
```

---

## Migration from v1.x

If you're using JSON backend:

```python
# Export from old version
memory_old = Memory(storage="json", path="./old_memory.json")
memory_old.export("backup.json")

# Import to new SQLite version
memory_new = Memory(storage="sqlite", path="./memory.db", ttl_days=30)
memory_new.import_("backup.json")
```

---

## Breaking Changes

- Default storage changed from `json` to `sqlite`
- TTL is now in days (not hours)
