The Problem: OpenHands Forgets Everything Between Sessions

65K stars, but every session starts from zero

OpenHands (formerly OpenDevin) is one of the most popular open-source AI coding agents. But like most AI agents, it has no persistent project memory. Every session begins with the agent re-discovering your codebase from scratch.

"OpenHands brings 65,000 GitHub stars as an open-source AI coding agent — But without persistent project memory, the agent re-discovers codebases every session." — MemU Blog, March 2026 (memu.pro)

This is a well-known limitation. When you start a new OpenHands session:

  • The agent doesn't remember your project's architecture
  • Coding conventions and style rules are forgotten
  • Previous bug fixes and decisions are lost
  • The agent spends the first 10-20 minutes re-orienting
  • Every session requires re-explaining the codebase context

How to Give OpenHands Persistent Memory

agent-memory — Universal MCP Memory for OpenHands

agent-memory is an open-source MCP server that gives OpenHands persistent memory. It works by exposing memory tools (add, search, list, get) via the Model Context Protocol — which OpenHands supports natively.

# 1. Install agent-memory pip install agent-memory # 2. Run the MCP server (OpenHands connects via MCP) python -m agent_memory.mcp_server --storage json --path ./openhands-memory.json # 3. Configure OpenHands to use the MCP server # Add to OpenHands MCP config: # { "mcpServers": { "agent-memory": { "command": "python", "args": ["-m", "agent_memory.mcp_server"] } } } # 4. Now OpenHands remembers your project across sessions # agent_memory.add(\"Project uses PostgreSQL for users\") # agent_memory.search(\"database schema\")
