# GitHub Issue Signal Capture Template - 信号捕获模板

## 核心设计原则

每个 Issue 必须包含：

### 1. 结构化问题（必选）
```
### 🎯 请投票

- [ ] 我需要这个功能
- [ ] 我不需要，但觉得有价值
- [ ] 我不关心
```

### 2. 开放式问题（必选）
```
### 💬 请留言

如果你有其他想法，请在这里直接评论。
```

### 3. CTA 引导（必选）
```
---
### ⏰ 反馈时间

请在 48 小时内回复。逾期可能不会继续开发此功能。
```

### 4. 联系方式备选（可选）
```
📧 也可以邮件联系我们: example@email.com
```

---

## 模板 1: 功能验证 Issue

```markdown
## 功能验证: {Feature Name}

### 📋 这是什么
{简短描述功能}

### 🎯 请投票（单选）
- [ ] 我需要这个功能 - 会立刻使用
- [ ] 我不需要 - 不符合我的使用场景
- [ ] 观望 - 取决于价格/质量

### 💬 请留言（选填）
你希望这个功能怎么实现？有什么特别的需求？

### 🔗 相关链接
- 演示: {demo_url}
- 文档: {doc_url}

---
### ⏰ 反馈时间
请在 48 小时内回复。此功能将根据反馈决定开发优先级。

*请直接评论，不要只是点赞，感谢！*
```

## 模板 2: 问题收集 Issue

```markdown
## 问题收集: {Topic}

### 📋 背景
{描述背景}

### ❓ 请回答

1. 你最痛的点是: _______
2. 你现在怎么解决的: _______
3. 你愿意为解决方案付费吗: 是 / 否 / 看情况

### 💬 更多想说的
（请直接评论）

---
### ⏰ 你的声音很重要
每个回复都会影响产品方向。期待听到你的想法！
```

## 模板 3: Beta 测试征集 Issue

```markdown
## Beta 测试征集: {Feature Name}

### 📋 功能简介
{描述功能}

### 🏃 如何参与
1. 在本 Issue 评论 "我要参与"
2. 我们会私聊你测试版本
3. 测试后提供反馈

### 🎁 参与奖励
- 优先获得正式版
- 开发者直接沟通
- 影响产品方向

### ❓ 条件
- 有 {技术要求}
- 愿意花 {时间投入} 测试

---
### ⏰ 征集截止
本次征集截止到 {日期}，或满 {人数} 人为止。
```

## 模板 4: 定价调研 Issue

```markdown
## 定价调研: {Product/Feature}

### 💰 背景
{描述产品/功能}

### 💳 请选择

**如果这是付费功能，你愿意为以下哪个方案付费？**

- [ ] 免费版（基础功能）
- [ ] $5/月（标准功能）
- [ ] $15/月（高级功能）
- [ ] 我只想要免费版
- [ ] 我不愿意付费

### 💬 请留言
你为什么做出这个选择？你期望的定价是多少？

---
### 📊 调研目的
此定价将根据反馈确定。真实数据将决定产品方向。
```

## 使用方法

1. 选择合适的模板
2. 填入 {} 中的内容
3. 使用 github_issue.py 脚本发布

```bash
python3 scripts/github_issue.py \
  --title "[投票] 你需要 Web Dashboard 吗？" \
  --body "$(cat templates/signal_capture_feature.md)" \
  --label "question,signal-capture"
```

## 信号跟踪规则

### 每天检查
1. 打开 https://github.com/lucksheep1/Repository-name-ai-saas-lab/issues
2. 查看每个带 "signal-capture" 标签的 Issue
3. 检查新评论、新 reaction

### 记录格式
| Issue # | 日期 | 新评论数 | 关键信息 | 行动 |
|---------|------|---------|---------|------|
| #3 | 03-18 | 0 | - | 继续等待 |
| #4 | 03-18 | 1 | "需要Dashboard" | 记录 |

### 信号判定
- **强信号**: 直接评论、具体需求
- **中信号**: reaction、emoji
- **弱信号**: 仅仅查看（无法追踪）
