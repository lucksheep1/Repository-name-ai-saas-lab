# MEMORY.md - 长期记忆

本文件用于沉淀跨 session 的长期信息。区别于 memory/YYYY-MM-DD.md（日常过程记录），这里只存"跨时间仍然有效"的内容。

## 写入触发条件

满足以下任一条件时，写入 MEMORY.md：

1. **用户明确偏好** — 用户告诉我"以后都用这种方式沟通"类偏好
2. **协作边界决策** — 我和用户达成明确的工作边界共识
3. **已运行的实证结论** — 经过实际运行验证的结论。排除：
   - 单纯讨论未执行的共识
   - 一次未报错就当作有效
   - 用户口头同意但未实际使用
4. **反复失败的教训** — 同一类错误犯过2次以上，必须记录
5. **重大方向决策** — 涉及项目/策略/资源分配的长期决策

## 写入格式

每条记录使用统一格式：

```
### [时间] 主题

**触发场景**: （简述为什么写这条）
**核心内容**: （核心结论，1-2句话）
```

示例：

```
### 2026-04-02 协作模式约定

**触发场景**: 用户要求区分讨论与执行模式
**核心内容**: 用户给的任务没有明确模式时，默认"只读分析"
```

## 不写入内容

以下内容**不**写入 MEMORY.md，应放在 memory/YYYY-MM-DD.md：

- 一次性会话的细节
- 临时分析过程
- 未验证的猜想
- 情绪化表达
- 短期可过期信息
- 可以从文件中直接查到的内容

## 触发式检查

- 每当我在新 session 中发现"这个问题之前讨论过但没记录"时，补充写入
- 不主动设周期回顾，避免因忘记而导致挫败感
