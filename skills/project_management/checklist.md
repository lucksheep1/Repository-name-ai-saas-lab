# Project Management Checklist

## 每轮执行检查清单

### Scout 阶段
- [ ] 扫描趋势/痛点 (GitHub Issues, Trending, HN)
- [ ] 产出 `trends/trend_report.md`
- [ ] 识别 3-8 个候选机会

### Scanner 阶段
- [ ] 验证机会 (≥3 条证据)
- [ ] 产出 `opportunities/opportunity_report.md`
- [ ] 产出 `opportunities/competitor_analysis.md`
- [ ] 满足机会 Gate (高频/强烈/可实现/可验证/正向价值)

### Builder 阶段
- [ ] MVP ≤4 小时, ≤500 行核心代码
- [ ] 产出 `projects/<name>/README.md`
- [ ] 产出 `projects/<name>/experiment_log.md`
- [ ] 产出 `projects/<name>/value_score.md`

### Analyst 阶段
- [ ] 评估 Pain/Frequency/Market/Competition/Differentiation
- [ ] 决策: Kill / Iterate / Scale
- [ ] 产出 `analysis/startup_analysis.md`

### Evolution 阶段
- [ ] 更新 `analysis/evolution_log.md`
- [ ] 更新 `analysis/scoring_weights.md`
- [ ] 更新 `opportunities/backlog.md`
- [ ] Git commit

---

## AM/PM 报告检查清单

### Evidence Delta (本轮新增证据)
- [ ] 列出所有新增证据
- [ ] 附链接或 commit permalink

### Skill Delta (本轮新增/更新技能)
- [ ] 列出新增 skills 条目
- [ ] 列出更新内容

---

## Scale Gate 检查清单

### Day 1
- [ ] 锁定主线项目
- [ ] 暂停其他项目
- [ ] 创建反馈入口

### Day 2
- [ ] 收集 ≥3 条证据
- [ ] 产出 Examples

### Day 3
- [ ] 验证证据数量
- [ ] 准备升级/降级决策
